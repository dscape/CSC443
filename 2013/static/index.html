            Database System Technology

            CSC443, Winter 2013

            1. Articles

            Description Programming Practical Learning
            Introduction About this course Setup your programming environment.
            Memory Hierarchy Memory hierarchy and their access characteristics Experiment with page based secondary storage data access. PL1
            Data Layout Storage and maintenance of relational data on secondary storage devices using heap file with pages. Implement a heap file to store relational data in pages on disk. star PL2
            Basic Index The concept of indexing and simple index structures Use your heap file implementation to speed up record search and scan. PL3
            B+ tree A very efficient disk-based data structure based on the idea of balanced sorted trees. Utilize B+ tree library (leveldb) to speed up record search and scan. Here is a quick tutorial on leveldb. PL4
            Hash index A very efficient disk-based data structure based on the ideas of extendible and linear hashing.
            Disk based sorting Sorting data on disk Implement the sorting algorithm with different parameters. PL5
            Keyword queries Searching text documents using approximate string matching Implement a search engine using the Xapian library. Here is a tutorial on programming using Xapian. PL6
            PROJECT: Application of database technology Extend and/or apply one of the physical database technology to solve a problem. Project

            4. Policy

            Text: Database Management Systems, by Raghu Ramakrishnan and Johannes Gehrke, 3rd Edition. The book's website has good resources including slides and answers to some of the problems in the book. Note: the bookstore originally had the wrong book, I'm told that has now been fixed but please double check that you get the right book.

            Marking: the practical learning tasks (PL1-PL6) and project can be complete in teams of up to 3 people. One of the practical learning tasks may be handed in late (with a 20% late penalty). Late tasks will not be marked until Week 11.

            Submission: all members of the team must submit using your account on CDF before the TA marks your work. Use "man submit" on CDF if you are not familiar with this command.

            Deadlines: all practical learning tasks must be submitted by Monday 5pm (on CDF) of the week indicated above. Meaning: Week 4 for PL1 (Monday Jan 28th 5pm), Week 6 for PL2, Week 7 for PL3 & PL4, Week 9 for PL5 & PL 6, Week 11 for project. Change announced Week 6: new submission time is Tues noon for PL2 onward, and PL4 now due Week 8, PL6 Week 10. Marking will be done interactively with the TAs in a tutorial after submission but no later than the tutorial of the weeks indicated.

            Tests: all tests permit one sheet of paper ("cheat sheet") on which you can write anything you like (front or back).

            Test Remark Requests: must be submitted in writing within two week of when the test was given. For the third test, requests must be submitted by April 8th 5pm.
